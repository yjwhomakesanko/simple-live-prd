[PRD] 살아있는 기획 시스템: liveprd

이 문서는 단순한 기록을 넘어, AI가 읽고 실제 코드를 생성하거나 시스템을 동기화하기 위한 **'기계 가독성(Machine-Readable)'**을 갖춘 설계 데이터입니다.

1. 배경 및 문제 의식 (The "Why")

1-1. 문서의 사후 관리 부재: 프로젝트 초기에만 정교하고, 이후 요구사항 변경 시 업데이트가 누락되어 문서가 '죽어버리는' 현상 발생.
1-2. 파편화된 정보: IA(엑셀), 정책(컨플루언스), 화면(피그마), 태스크(지라)가 따로 놀며 수동 동기화에 엄청난 에너지가 소모됨.
1-3. 맥락 소실: 결론만 남고 '왜' 그렇게 결정했는지 논리가 사라져 재검토 비용과 인수인계 난이도가 상승함.

2. 핵심 원칙 (The Manifesto)

2-1. 통합 (Integration): 모든 기획 요소는 유기적으로 연결된 단일 진실 공급원(SoT)을 지향함.
2-2. 구조화 (Structuring): 기획은 글쓰기가 아닌 '데이터 설계'임. (DB 스키마처럼 관리)
2-3. 기계 가독성 (Machine-Readable): AI가 즉시 파싱하고 추론하여 코드베이스와 대조할 수 있어야 함.
2-4. 버전 관리 (Versioning): 변경된 데이터 포인트만 추적하여 영향 범위를 파악함.
2-5. 검증 (Validation): 코딩 전 단계에서 논리적 결함과 예외 케이스를 AI가 선제적으로 체크함.

3. 데이터 계층 구조 (Information Architecture)

AI가 추론할 수 있도록 아래와 같은 엄격한 계층 구조를 유지함.

3-1. Level 1: Requirement (요구사항)

비즈니스 목표와 사용자 니즈 정의.
3-2. Level 2: Feature (기능)

요구사항을 충족하기 위한 시스템 단위의 기능 정의.
3-3. Level 3: Spec (상세 명세)

API 필드, 비즈니스 로직, 예외 케이스(Edge Cases), 구현 조건.
3-4. Level 4: User Flow (사용자 흐름)

시각적 레이아웃과 화면 전환 로직.

4. 핵심 기능 요구사항 (Functional Specs)

4-1. AI 에이전트 인터페이스 (Left Pane)

비정형 데이터(회의록, 대화) 입력 시 기존 문서와 대조 및 수정 제안.

변경 사항 발생 시 영향 범위(Impact Analysis) 리포트 자동 생성.
4-2. 인터랙티브 기획 뷰어 (Right Pane)

데이터 변경 시 실시간으로 업데이트되는 문서 뷰.

피그마(Figma) 연동을 통한 실시간 화면 시안 동기화.
4-3. 버전 및 이력 관리

결정 사항에 대한 논리적 근거 기록(Decision Log).

데이터 단위의 Snapshot 및 Rollback 기능.

5. 바이브코딩 연동 시나리오 (Implementation Flow)

5-1. 변경 감지: "결제 수단에 카카오페이 추가" 명령 입력.
5-2. 추론 및 업데이트:

Level 2(기능)에 '카카오페이' 추가.

Level 3(스펙)에 관련 API 정보 및 예외 케이스 생성.
5-3. GAP 분석: 현재 코드베이스를 스캔하여 미구현 항목 리포트 생성.
5-4. 코드 생성: PRD 데이터를 바탕으로 보일러플레이트 코드 및 테스트 코드 자동 생성.

6. 기술적 제약 및 규칙

6-1. No SVG/Image Labels: 모든 다이어그램과 구조는 AI가 이해하기 쉬운 텍스트/데이터 포맷으로 관리.
6-2. Strict Mapping: 모든 Spec은 반드시 상위 Feature와 Requirement에 연결되어야 함.
6-3. Validation First: 모든 변경사항은 '영향 범위 체크' 단계를 거친 후에만 최종 반영 가능.

참고: 이 문서는 지능형 기획 시스템의 근간이 되며, AI는 이 구조를 바탕으로 실제 애플리케이션의 로직을 설계해야 함.
